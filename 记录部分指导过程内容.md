## 1.
`XBYTE`。它不是 C 语言自带的，是 Keil 的头文件 **`absacc.h`**（access absolute，绝对地址访问）里定义的一个宏，原型大致是：

c

```c
#define XBYTE  ((unsigned char volatile xdata *) 0)
```

翻译成你熟悉的普通 C：**`XBYTE` 是一个指向 `unsigned char` 的指针，它的值（地址）是 0**。`xdata` 先忽略，等下讲。

那么 `XBYTE[0x7C00]` 就是最普通的数组下标语法，等价于：

c

```c
*(XBYTE + 0x7C00)     // 即：访问地址 0x7C00 处的那个字节
```

因为 `XBYTE` 从地址 0 起算、每个元素 1 字节，加 `0x7C00` 正好落在地址 `0x7C00`。

所以 `#define PA8255 XBYTE[0x7C00]` 这行**只是起了个别名**。以后你写：

c

```c
PA8255 = 0x38;        // 展开成 *(XBYTE+0x7C00) = 0x38;
```

意思就是：**往外部地址 0x7C00 写一个字节 0x38**。

## 2.

**`xdata` / `data` / `code` 这类"内存类型"关键字。** 8051 的内存被切成好几块独立空间（片内 RAM、片外 RAM、程序 ROM），普通 C 没有这回事。`xdata` 指"片外数据空间"（外部 64KB，8255 就挂在这），`data` 指片内直接寻址 RAM，`code` 指程序存储区（放常量表）。你现在只需知道 `XBYTE` 走的是 `xdata`，对应汇编的 `MOVX` 指令。

**2. `sfr` 和 `sbit`。** 在 `reg51.h` 里，`P1`、`TMOD`、`TH0` 这些寄存器名都是用 `sfr` 定义的特殊功能寄存器；`sbit` 让你按单个引脚操作，比如：

c

```c
sbit LED = P1^0;   // 给 P1 的第0位起名 LED
LED = 0;           // 直接拉低这一个引脚
```

你 DS1302、24C02、按键那些单线控制就靠 `sbit`。

**3. 中断函数有专门写法：**

c

```c
void T0_ISR(void) interrupt 1 { ... }   // interrupt 1 = 定时器0中断
```

`interrupt n` 是普通 C 没有的关键字，编号固定（Timer0 是 1）。

**4. `main` 永不返回。** 单片机没有操作系统，程序就是死循环，`main` 里最后一定是 `while(1){...}`，不会 `return`。

**5. 类型尺寸不一样。** 8051 上 `int` 是 **16 位**、`char` 是 8 位。实战里主力类型是 `unsigned char`（8 位，正好一个字节 / 一个端口），能用它就别用 `int`，省 RAM 又快。还有个 1 位的 `bit` 类型可以存标志。

**6. 别用 `malloc`、别用 `printf`（除非专门配串口）、延时靠空循环或定时器**，这些 PC 上的习惯这里基本用不上。

## 3.
### 一、送数的物理过程（LM016L 怎么"收货"）

LM016L 靠三根控制线 + 8 根数据线工作。送一个字节，硬件上要走这么个时序：

```
1) RS 选好：送数据=1，送指令=0
   (RW 我们接了 GND，恒为0=写，不用管)
2) 把这个字节放到 D0~D7（也就是 8255 的 PA 口）
3) E 拉高
4) E 拉低  ← LM016L 在 E 的【下降沿】把数据锁进去
```

**关键一句话：数据是在 E 从 1 变 0 的那一刻被读走的。** 所以顺序必须是"先把数据和 RS 摆好，E 再高→低"。数据在 E 拉低前必须是稳定的。

对照你的接线：

- 数据 D0~D7 → `PA8255`（0x7C00）
- RS → PC0，E → PC1 → 合在 `PC8255`（0x7E00）里

PC 口只用两位，直接写整个字节最省事：

|想要|RS(PC0)|E(PC1)|写 PC8255|
|---|---|---|---|
|送指令 · E高|0|1|`0x02`|
|送指令 · E低|0|0|`0x00`|
|送数据 · E高|1|1|`0x03`|
|送数据 · E低|1|0|`0x01`|

### 二、动作顺序（你按这个拼代码）

`Lcd_WriteData(dat)` 要做的：

```
PA8255 = dat;      // ① 数据先摆上总线
PC8255 = 0x03;     // ② RS=1(数据), E=1  → E 拉高
（短延时，稳一下）
PC8255 = 0x01;     // ③ RS=1 保持, E=0   → E 下降沿，LM016L 锁数据
（留点时间给它处理）
```

`Lcd_WriteCmd(cmd)` 一模一样，只是 RS=0，那两个值换成 `0x02` → `0x00`。

## 4.
**LM016L**规定的编码
- `0x80 | 地址` 整体是发给 `Lcd_WriteCmd` 的一条**命令**，作用是"把光标移到某个位置"。
- 它本身不显示任何东西，只是**定位**。定位完之后，你再用 `Lcd_WriteData` 送的字符才会出现在那个位置。

指令 `0x80 | addr` 里，`0x80` 是"设地址"这条指令的固定标志位（最高位=1），低 7 位 `addr` 才是**目标格子的地址**。

### 二、地址不是"第几行第几列"，是 LM016L 规定的编码

你想去"第二行第二格"，但地址**不能**用行列直接算成 `0x11`。LM016L（HD44780）的地址是**写死的**：

||第1列|第2列|第3列|…|第16列|
|---|---|---|---|---|---|
|**第一行**|0x00|0x01|0x02|…|0x0F|
|**第二行**|**0x40**|**0x41**|0x42|…|0x4F|

**关键点：第二行不是接着第一行的 0x10，而是从 `0x40` 突然跳过去。** 这是硬件规定，记住就行。

所以：

- 第一行第一格 = `0x00` → 指令 `0x80 | 0x00 = 0x80`
- 第二行第一格 = `0x40` → 指令 `0x80 | 0x40 = 0xC0`
- **第二行第二格 = `0x41`** → 指令 `0x80 | 0x41 = 0xC1`

## 5.
**DS1302

8255 是并行——8 根线一次送一个字节。DS1302 只有 **1 根数据线 I/O**，只能**一位一位**地挤出去/读进来，靠 SCLK 时钟"打拍子"同步。

发一个字节 = 循环 8 次，每次：把 1 个 bit 放上 I/O 线 → 给 SCLK 一个脉冲 → 对方在这一拍把这个 bit 收走。

**两条铁规矩（DS1302 datasheet 定的）：**

1. **LSB 先出**：先发最低位 bit0，最后发 bit7。（跟人的直觉相反，注意！）
2. **写数据**：DS1302 在 SCLK **上升沿**（0→1）读走 I/O 上的位。  
    **读数据**：DS1302 在 SCLK **下降沿**后把位放到 I/O 上，你在低电平期间读。
### 通信的完整流程

要读/写 DS1302 一个寄存器，整体是：

```
CE 拉高（开始）
  → 发 1 字节【命令/地址】（告诉它：我要读还是写、操作哪个寄存器）
  → 再 发/收 1 字节【数据】
CE 拉低（结束）
```

**命令字节**的格式（8 位）：

```
 bit7  bit6   bit5~1        bit0
  1   RAM/CK  地址          RD/WR
  |     |                    |
 固定  0=时钟寄存器      1=读, 0=写
```

好在你不用手拼——我们之前指导书给了现成地址：**读年=0x8D、读月=0x89、读日=0x87**（这些已经是"命令字节"，bit0=1 表示读）。

## 6.
DS1302 里数据完全正确（`12-00-46`、`09-07-26`），但屏上是 `25:01:0=`、`204=-0?-13`——**数字大体对得上、但夹杂着 `=` `?` `-` 这种乱字符**。这不是进制问题了，是**读时序偶尔出错，读回了错误的字节**。

### 一、先看证据：错的是"个别位"，不是"全错"

对比一下（真实值 vs 屏显）：

```
真实 12:00:46      屏显 25:01:0=
真实 2026-07-09    屏显 204=-0?-13
```

你会发现：**有些位对、有些位跳成乱码**，而且每次刷新（0.1s）都在变。如果是进制错，应该是**稳定地错成固定规律**；现在是**飘忽不定的乱码**——这是典型的**通信不稳定 / 读到错位的数据**。

字符 `=` `?` 是怎么来的？`(x>>4)+'0'`，当 `x` 的高半字节 > 9（比如 0xD、0xE），`+'0'` 就超出 '0'~'9' 变成 `:;<=>?` 这些 ASCII——**说明读回的字节根本不是合法 BCD**，被污染了。

### 二、最可能的原因：一次刷新里读了太多次，时序打架

你现在 **0.1s 刷一次**，每次 `ShowTime`+`ShowDate` 要连续调用 **7 次 `DS1302_Read`**（时分秒 + 日月年周）。每次 Read 都在快速拉 CE/SCLK。DS1302 是慢速器件，这么密集地连续单字节读，**相邻两次通信之间没留够间隙**，容易串位——读到一半上一次的余波还没稳。

再加上你 `DS1302_WByte` / `RByte` 里**时钟翻转之间一个延时都没有**（`DS_SCLK=1; DS_SCLK=0;` 紧挨着），在 6MHz 下这两条指令间隔可能太快，DS1302 跟不上，偶尔就读错一位。

### 三、两个改法，从简单到彻底

#### 改法 1（先试这个）：给 SCLK 加一点点延时

在 `WByte` 和 `RByte` 的时钟翻转之间插一个极短延时，让 DS1302 跟得上。加个 `_nop_()`：

c

```c
void DS1302_WByte(unsigned char dat){
    unsigned char i;
    for(i=0;i<8;i++){
        DS_IO = dat & 0x01;
        DS_SCLK = 0;
        _nop_();                 // ← 加
        DS_SCLK = 1;
        _nop_();                 // ← 加
        dat >>= 1;
    }
}

unsigned char DS1302_RByte(void){
    unsigned char i, dat=0;
    for(i=0;i<8;i++){
        dat >>= 1;
        if(DS_IO) dat |= 0x80;
        DS_SCLK = 1;
        _nop_();                 // ← 加
        DS_SCLK = 0;
        _nop_();                 // ← 加
    }
    return dat;
}
```

#### 改法 2（更该做）：别 0.1s 狂刷，降到每 0.5s 刷一次

你 0.1s 刷新一次纯属浪费——秒最多 1 秒变一次，0.1s 刷 10 次没意义，还徒增 DS1302 通信次数、增加出错机会。改成**每 5 次中断（0.5s）才刷一次**：

c

```c
// ISR 里
if(++ISR_1s >= 5){       // 0.1s × 5 = 0.5s
    ISR_1s = 0;
    bRefresh = 1;        // 0.5s 才置一次刷新
}
```

（把 `bRefresh=1` 从 ISR 顶部挪进这个 `if` 里，别每次中断都置位。）

这样 DS1302 每 0.5s 才被读一轮，通信压力小很多，稳定性大增，秒看起来照样跳。

#### 改法 3
### 稳定的 DS1302 驱动

别在细节上耗了，用这版被无数人验证过的写法，关键是**每次操作后 SCLK 明确回到 0**、**读前释放 I/O**：

c

```c
void DS1302_WByte(unsigned char dat)
{
    unsigned char i;
    for (i = 0; i < 8; i++)
    {
        DS_IO = dat & 0x01;
        _nop_();
        DS_SCLK = 1;          // 上升沿写入
        _nop_();
        DS_SCLK = 0;          // ★ 回到0，为下一位/读做准备
        dat >>= 1;
    }
}

unsigned char DS1302_RByte(void)
{
    unsigned char i, dat = 0;
    DS_IO = 1;                // ★ 读前释放I/O为输入
    for (i = 0; i < 8; i++)
    {
        if (DS_IO) dat |= (1 << i);   // 先读(LSB先进，第i位)
        DS_SCLK = 1;
        _nop_();
        DS_SCLK = 0;                   // 下降沿取下一位
        _nop_();
    }
    return dat;
}
```

注意两处关键改动：

1. **`WByte` 每位以 `SCLK=0` 收尾**（你原来收尾是 `SCLK=1`，这是相位错乱、乱跳的根源）。
2. **`RByte` 开头 `DS_IO=1` 释放输入**，且用 `1<<i` 直接按位放（比 `>>=` 更直观不易错）。

### 为什么这能解释"乱跳"

你原来 `WByte` 结尾 SCLK 停在 1，导致命令字节和数据字节之间的时钟边沿**多了或少了一个**，DS1302 有时对齐、有时错位 → 读回的字节**时对时错**→ 屏上乱跳。SCLK 每位规规矩矩回 0，相位就稳了。

### 复盘一下这个 bug（值得记进报告）

这是你整个课设**最有技术含量的一个 debug**，答辩很可能被问，记住这条因果链：

- **现象**：数字大体对但夹杂 `= ? -` 乱码，且每次刷新乱跳。
- **误判方向**：一开始以为是进制转换 / 硬件 / 晶振——但"乱跳"这个特征其实早就排除了静态错误（进制错会**稳定错**，不会跳）。
- **根因**：`DS1302_WByte` 每位以 `SCLK=1` 结尾，退出时时钟停在高电平，导致后续读时序的时钟边沿错位一拍，读回的字节间歇性错位。
- **修复**：每位以 `SCLK=0` 收尾 + 读前 `DS_IO=1` 释放准双向口为输入。

一句话总结：**"乱跳"= 动态/时序问题，"稳定错" = 静态/逻辑问题**。以后 debug 先按这个分方向，能少走弯路

## 7.
**为什么"按下是低电平"**：8051 的 P3 口内部有上拉，平时读到高电平(1)；按键另一端接地，按下时把引脚拉到地(0)。所以**按下 = 读到 0**。

Proteus 里：搜 `BUTTON`，一端接 P3.2/P3.3，另一端接 **GND**。

sbit 定义：

c

```c
sbit KEY_CLR = P3^3;   // 清除
sbit KEY_SS  = P3^2;   // 启停
```

### 三、两个功能怎么实现（用你已有的 DS1302 函数）

**清除** = 把时间写成 00:00:00：

c

```c
// 写秒/分/时都为0（记得先解写保护）
DS1302_Write(0x8E, 0x00);   // 解写保护
DS1302_Write(0x80, 0x00);   // 秒=0 (CH=0继续走)
DS1302_Write(0x82, 0x00);   // 分=0
DS1302_Write(0x84, 0x00);   // 时=0
DS1302_Write(0x8E, 0x80);   // 上写保护
```

**启停** = 翻转秒寄存器的 **CH 位**（bit7）。这就是当初讲 DS1302 时埋下的伏笔，现在真用上了：

c

```c
// 读秒寄存器 → 翻转bit7 → 写回
unsigned char sec = DS1302_Read(0x81);   // 读回的bit7就是CH
sec ^= 0x80;                             // 异或0x80 = 翻转最高位
DS1302_Write(0x8E, 0x00);                // 解写保护
DS1302_Write(0x80, sec);                 // 写回(CH位已翻转)
DS1302_Write(0x8E, 0x80);                // 上写保护
```

- CH=1 → 时钟停走，屏上时间定住
- CH=0 → 继续走
- `sec ^= 0x80` 是"异或翻转"：`^0x80` 让 bit7 在 0/1 之间切换，其他位不变。

> ⚠️ 注意：翻转时要保留秒的低7位（当前秒值），所以是"读出来改bit7再写回"，不能直接写 0x80——那样会把秒也清了。你的 `sec ^= 0x80` 正好只动 bit7，对的。

### 四、去抖 + 扫描的框架（你来补全）

在主循环里加一个按键扫描。框架给你，`?` 和函数体你写：

c

```c
void Key_Scan(void)
{
    if (KEY_CLR == 0)              // 检测到清除键按下(低电平)
    {
        delay_ms(20);             // 去抖：等20ms
        if (KEY_CLR == 0)         // 再确认一次，还是按下才算数
        {
            // TODO: 执行"清除"——写 00:00:00
            while (KEY_CLR == 0); // 等松手(防止按住不放连续触发)
        }
    }

    if (KEY_SS == 0)              // 启停键
    {
        delay_ms(20);
        if (KEY_SS == 0)
        {
            // TODO: 执行"启停"——翻转CH位
            while (KEY_SS == 0);  // 等松手
        }
    }
}
```

主循环加上调用：

c

```c
while (1)
{
    Key_Scan();                   // 每圈都扫一次按键
    if (bRefresh) {
        bRefresh = 0;
        Lcd_ShowTime();
        Lcd_ShowDate();
    }
}
```

### 五、去抖的三段式（记住这个套路）

```
1. if(键==0)          → 发现可能按下
2. delay 20ms + 再判断 → 确认不是抖动
3. while(键==0);      → 等松手，保证一次按下只触发一次
```

这三段是按键处理的**万能模板**，阶段⑤的换挡/加数键也用它。

## 8.
逻辑思路全对，尤其**设秒时你主动保住了 CH 位**（`sec_top |= 0x80`）——这是很细心的一步，说明你真的理解了"设置状态下时钟是暂停的、写秒不能把 CH 丢了"。但有几个具体问题，一个是编译错、两个是隐患。

### 🔴 编译错：设秒那行漏了分号

c

```c
sec_top |= 0x80;
```

`Dec_To_BCD(sec)` 后面**漏了 `;`**，改成：

c

```c
unsigned char sec_top = Dec_To_BCD(sec);   // 加分号
```

### 🔴 隐患：设秒时读秒没屏蔽 CH 位

设秒时你读的是 `DS1302_Read(DS_SEC_R)`（0x81），但**此刻时钟是暂停的，CH=1**，所以读回的 bit7=1。你直接丢给 `BCD_To_Dec`：

c

```c
unsigned char sec = BCD_To_Dec(DS1302_Read(DS_SEC_R));  // 读回带CH位!
```

假设当前秒是 5、CH=1，读回 `0x85`，`BCD_To_Dec(0x85)` = 8×10+5 = **85**！秒瞬间变成 85，再 +1、判 60 归零……全乱。

**修法**：读秒时先屏蔽 CH 位：

c


### 正确的写法（大幅简化）

**第一步：ISR 改成翻转**

c

```c
void T0_ISR(void) interrupt 1
{
    TH0 = 0x3C;
    TL0 = 0xB0;
    if (++ISR_1s >= 2) {        // 0.2s 刷新
```

顶部全局：`unsigned char blink_cnt = 0; volatile bit Blink = 1;`

```c

{
    if (hide) { Lcd_WriteData(' '); Lcd_WriteData(' '); }
    else { Lcd_WriteData((bcd>>4)+'0'); Lcd_WriteData((bcd&0x0F)+'0'); }
```

**第三步：`Lcd_ShowTime` 用它——整个函数缩成这样**

```c
void Lcd_ShowTime(void)
{
    unsigned char h, m, s;
    h = DS1302_Read(DS_HOUR_R);
    m = DS1302_Read(DS_MIN_R);
    s = DS1302_Read(DS_SEC_R) & 0x7F;

    Lcd_WriteCmd(0x80);
    Lcd_Show2(h, (set_mode==1) && (Blink==0));   // 设时且灭阶段→空格
    Lcd_WriteData(':');
    Lcd_Show2(m, (set_mode==2) && (Blink==0));   // 设分
    Lcd_WriteData(':');
    Lcd_Show2(s, (set_mode==3) && (Blink==0));   // 设秒
}
```

- 你的：3 个分支、30+ 行、`Blink` 被清 0（bug）

- 没设置（set_mode=0）→ 条件恒 false → 全部正常显示，不闪
- 设时（set_mode=1）→ 时那位跟 Blink 亮灭闪，分秒正常

## 10.
### 四个基本信号（一帧通信的积木）

#### 1. 起始信号 Start

**定义：SCL 高电平期间，SDA 由高→低跳变。**

```
SCL ────────────
SDA ─────┐
         └──────   ← SCL还是高时，SDA下降 = Start
```

这是唯一允许 SDA 在 SCL 高时变化的两个时刻之一（另一个是 Stop）。平时数据位只能在 SCL 低时变。Start 一发，所有从机都竖起耳朵。

你的代码：

c

```c
void I2C_Start(void){
    I2C_SDA = 1; I2C_SCL = 1; _nop_();   // 先都拉高
    I2C_SDA = 0; _nop_();                // SCL高时SDA下降 ← Start
    I2C_SCL = 0;                         // 拉低SCL,准备发数据
}
```

#### 2. 发送一个字节（8 位，MSB 先）

数据位规则：**SCL 低时改变 SDA，SCL 高时从机采样**。所以发一位是：

```
SCL低 → 把这一位放上SDA → SCL高(让对方读) → SCL低
```

重复 8 次，**最高位先发**（MSB first，和 DS1302 的 LSB 相反！）：

c

```c
for(i=0;i<8;i++){
    I2C_SDA = (dat & 0x80);   // 取最高位
    I2C_SCL = 1; _nop_();     // 拉高,从机采样
    I2C_SCL = 0;
    dat <<= 1;                // 左移,下一位变最高位
}
```

#### 3. 应答信号 ACK

**每发完 8 位，第 9 个时钟是"应答位"**——主机松开 SDA，从机把 SDA 拉低表示"我收到了"。

```
主机发完8位 → 释放SDA(=1) → 给第9个时钟 → 读SDA
  SDA=0 → ACK  (从机应答,收到了)
  SDA=1 → NACK (没应答,可能没这个设备/出错)
```

c

```c
I2C_SDA = 1;              // 释放SDA,交给从机
I2C_SCL = 1; _nop_();
ack = I2C_SDA;            // 读:0=应答 1=无应答
I2C_SCL = 0;
```

**ACK 的意义**：它是握手确认。如果你发器件地址后读到 NACK，说明总线上没有这个地址的设备（地址错、没接、上拉没接都可能）。调试时可以靠它判断"设备在不在"。

#### 4. 停止信号 Stop

**定义：SCL 高电平期间，SDA 由低→高跳变。**（和 Start 正好相反）

```
SCL ────────────
SDA        ┌─────
      ─────┘        ← SCL高时,SDA上升 = Stop
```

c

```c
void I2C_Stop(void){
    I2C_SDA = 0; I2C_SCL = 1; _nop_();   // SDA先低,SCL高
    I2C_SDA = 1; _nop_();                // SCL高时SDA上升 ← Stop
}
```

Stop 一发，总线释放，回到空闲。

## 11.
### 正确逻辑：Init 只做一次

设定初始时间这件事，**逻辑上只该在"第一次"做**——出厂/第一次烧录时设一次基准，之后就让 DS1302 自己走，永远别再 `Init`。

怎么判断"是不是第一次"？——**正是用 24C02 的魔术字**！这两个功能在这里完美配合：

```
上电 → 读 24C02 魔术字
  ├─ 有魔术字(0x5A) → 不是第一次 → 【不调 DS1302_Init】→ 信任 DS1302 保持的时间
  └─ 没魔术字      → 第一次上电 → 【调 DS1302_Init 设 12:00】→ 写入魔术字
```

所以修法就是把 `DS1302_Init()` 从 main 挪到 `Restore_FromE2` 的**首次分支**里：

c

```c
void Restore_FromE2(void)
{
    unsigned char magic = E2_Read(E2_MAGIC);
    if (magic == MAGIC_VALUE)        // 不是首次
    {
        boot_count = E2_Read(E2_COUNT) + 1;
        if (E2_Read(E2_RUNSTATE) == 1) Stop_DSTime();
        // 不调 Init,信任 DS1302 的时间
    }
    else                             // 首次!
    {
        DS1302_Init();               // ★只有这里才设初值
        E2_Write(E2_MAGIC, MAGIC_VALUE);
        boot_count = 1;
    }
    E2_Write(E2_COUNT, boot_count);
    Backup_ToE2();
}
```

main 里删掉那句 `DS1302_Init();`。
