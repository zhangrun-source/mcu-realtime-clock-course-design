# 调试说明：基于 8031 的多功能实时钟

本文记录项目联调中出现的问题、定位方法和最终修复方案。开发环境为 Keil C51，仿真环境为 Proteus；主控为 8031，外设包括 8255A、LCD1602、DS1302 和 24C02。

## 1. 调试顺序

联调按“先能编译、再看通信、最后看显示和交互”的顺序进行：

1. Keil 编译通过，确认生成的 HEX 文件可加载。
2. 在 Proteus 中确认 8031 时钟、复位和 8255A 地址线连接正确。
3. 单独验证 DS1302 的读写时序，再显示时间和日期。
4. 最后加入按键设置、闪烁提示和 24C02 状态保存。

这样可以避免把显示、通信和按键问题混在一起排查。

## 2. DS1302 读数偶发乱码

### 现象

DS1302 调试窗口中的时间数据正常，但 LCD1602 偶尔显示异常字符，且每次刷新时内容会变化。数字并非始终错误，因此不像 BCD 转换公式或硬件接线的静态错误。

### 定位

检查 `DS1302_WByte` 和 `DS1302_RByte` 后，发现写一个位后的时钟电平没有稳定回到低电平。命令字节与数据字节之间可能多出或缺少一个边沿，造成后续读回数据偶发错位。

同时，DS1302 的 I/O 为双向线。读取前若未释放 I/O，单片机与 DS1302 可能同时驱动该线路，读取结果也会不稳定。

### 修复

写操作中每一位都以 `SCLK = 0` 收尾；读操作开始前释放 I/O，并按 LSB 优先的顺序收集数据位：

```c
void DS1302_WByte(unsigned char dat)
{
    unsigned char i;
    for (i = 0; i < 8; i++) {
        DS_IO = dat & 0x01;
        _nop_();
        DS_SCLK = 1;
        _nop_();
        DS_SCLK = 0;
        dat >>= 1;
    }
}

unsigned char DS1302_RByte(void)
{
    unsigned char i, dat = 0;
    DS_IO = 1;
    for (i = 0; i < 8; i++) {
        if (DS_IO) dat |= (1 << i);
        DS_SCLK = 1;
        _nop_();
        DS_SCLK = 0;
        _nop_();
    }
    return dat;
}
```

修复后，时间和日期连续刷新不再出现乱码。该问题说明：显示内容“乱跳”通常优先检查动态时序；若显示内容稳定但数值错误，再检查地址、BCD 转换和业务逻辑。

## 3. 刷新过于频繁

### 现象与定位

定时器每 0.1 秒触发一次刷新标志。秒值每秒才变化一次，过于频繁地读取 DS1302 增加了总线通信次数，也放大了时序不稳定时的现象。

### 修复

保留 0.1 秒的定时器节拍，但每 5 次中断才请求一次显示刷新：

```c
if (++ISR_1s >= 5) {
    ISR_1s = 0;
    bRefresh = 1;
}
```

主循环仅在 `bRefresh` 置位后调用 `Lcd_ShowTime()` 与 `Lcd_ShowDate()`。这样仍能保持界面响应，同时将 DS1302 读取频率降为每 0.5 秒一次。

## 4. 设置秒时的编译与状态问题

### 问题

设置秒的分支曾出现漏写分号，导致 Keil 编译报错。另一个隐患是直接使用秒寄存器的完整字节参与计算：DS1302 秒寄存器的 bit7 为 CH（Clock Halt）位，不是秒值。

### 修复

读取秒数时使用 `& 0x7F` 屏蔽 CH 位；写回时保留原有的 CH 位。这样设置秒数不会意外改变时钟启停状态。

```c
unsigned char sec_raw = DS1302_Read(DS_SEC_R);
unsigned char ch = sec_raw & 0x80;
unsigned char sec = BCD_To_Dec(sec_raw & 0x7F) + 1;
if (sec >= 60) sec = 0;
DS1302_Write(DS_SEC_W, ch | Dec_To_BCD(sec));
```

## 5. 按键重复触发

### 现象与定位

四个按键均采用低电平有效。机械按键按下和松开时会抖动，若直接在主循环判断，会出现一次按下加多次或模式连续跳转的情况。

### 修复

对每个按键使用“检测—延时确认—等待松开”的三段式处理：

```c
if (KEY_ADD == 0) {
    delay_ms(20);
    if (KEY_ADD == 0) {
        /* 执行一次加值操作 */
        while (KEY_ADD == 0);
    }
}
```

经 Proteus 连续点击验证后，每次按下只触发一次，设置模式、加值、清零与启停功能均正常。

## 6. 显示闪烁逻辑

设置时、分、秒时，需要只闪烁当前字段。将闪烁状态作为只读条件传给显示函数，而不在显示函数内修改 `Blink`，避免显示过程影响定时器维护的闪烁节奏：

```c
Lcd_Show2(hour, (set_mode == 1) && (Blink == 0));
Lcd_Show2(min,  (set_mode == 2) && (Blink == 0));
Lcd_Show2(sec,  (set_mode == 3) && (Blink == 0));
```

## 7. 最终验证

- Keil C51 可生成 `keil_c/Objects/c1.hex`。
- Proteus 中 DS1302 时间、日期与 LCD1602 显示一致。
- 清零、启停、模式切换和加值按键均能单次触发。
- 24C02 的 I2C 连线带上拉电阻，状态保存流程可正常工作。
